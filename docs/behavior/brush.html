brush

  1. - behavior in module wk.chart

enable brushing behavior

Directive Info

  • This directive executes at priority level 0.

Usage

  • as attribute: 
    <x, y, range-x, range-y or layout
  [brush=""]
  [brush-extent=""]
  [selected-values=""]
  [selected-domain=""]
  [selected-domain-change=""]
  [brush-start=""]
  [brush-end=""]
  [clear-brush=""]>
...
</x, y, range-x, range-y or layout>

Arguments

Param Type Description
brush
(Accepted Values: none)
 string

Brush name Brush will be published under this name for consumption by other layouts
brushExtent
array

Contains the start and end index into the data array for the brushed axis. Is undefined if brush is empty or is a xy (layout) brush
selectedValues
array

Contains array of the axis values of the selected the brush area. Is undefined if brush is empty or is xy (layout) brushes
selectedDomain
array

Contains an array of data objects for the selected brush area.
selectedDomainChange
expression

expression to evaluate upon a change of the brushes selected domain. The selected domain is available as ´domain´
brushStart
expression

expression to evaluate upon a start of brushing. Is fired on 'mousedown'.
brushEnd
expression

expression to evaluate upon a end of brushing. is fired on 'mouseup'. The selected domain is available as ´domain´
clearBrush
function

assigns a function that clears the brush selection when called via a bound scope variable. Usage: bind a scope variable to the attribute: clear-selection="scopeVar". brush assigns a function to scopeVar that can be called to reset the brush, e.g. in a button: <button ng-click="scopeVar()">Clear Brush</button>
